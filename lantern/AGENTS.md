# `lantern/` — R Package

R package for ancestry-specific rare variant association analysis with a C backend. v0.1.0, License MIT.

## LAYOUT

```
lantern/
├── DESCRIPTION / NAMESPACE        # package metadata
├── R/
│   ├── lantern.R                  # C wrappers + exported R API
│   └── workflow.R                 # run_ancestry_pipeline() (overlap handling, monomorphic filter)
├── src/                           # C backend — see lantern/src/AGENTS.md
│   ├── ancestry.c                 # split_by_ancestry, count_ancestry_codes
│   ├── ancestry.h                 # C function declarations
│   ├── init.c                     # R_CallMethodDef registration
│   └── Makevars                   # PKG_CFLAGS = -O3 -march=native -Wall
└── tests/testthat/
    ├── test-functions.R           # count_ancestry_codes + split_by_ancestry
    └── test-workflow.R            # run_ancestry_pipeline (overlap, monomorphic)
```

## BUILD & TEST

Always go through pixi (system gcc 9.3 breaks R 4.5 C23 enum syntax):

```bash
pixi run install-lantern          # R CMD INSTALL lantern/
pixi run test                     # devtools::test("lantern")
```

If the install fails with `Rboolean` errors, your `~/.R/Makevars` is being used instead of `R.Makevars.local`. Confirm `R_MAKEVARS_USER=$PWD/R.Makevars.local` is set in the task.

## CONVENTIONS (package-specific)

- **C entry registration**: every C function listed in `init.c` MUST also have a matching declaration in `ancestry.h` AND a non-`static` definition in `ancestry.c`. Mismatch causes linker/compiler errors.
- **R exports**: add an `@export` roxygen tag in `R/lantern.R` or `R/workflow.R`, then re-run roxygen. Don't hand-edit `NAMESPACE` (it says "Generated by roxygen2: do not edit by hand").
- **Matrix types**: genotype and PT matrices must be `integer` storage. R wrappers coerce via `storage.mode() <- "integer"`.
- **Ancestry codes**: integer 1/2/3, not strings, not factors.
